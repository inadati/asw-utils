# wantree スキル

## 概要

`want` と `tree` の2フィールドだけで要件を階層的に整理するYAMLフォーマット（wantree）を
ヒアリング形式で対話的に作成・管理するスキル。

音声入力での呼び出しを想定:
  wantree → 「ワントリー」「ウォントリー」「ワントゥリー」など

## wantree.yml フォーマット

```yaml
want: ここに目的・要望を書く
tree:
  - want: サブ要件1
  - want: サブ要件2
    tree:
      - want: さらに詳細
```

## 適用条件とモード判定

起動時にカレントディレクトリの `.wantree/` の状態を確認し、モードを判定する:

- **`.wantree/` が存在しない** → 初期化してヒアリング開始
- **`.wantree/` が存在する** → 一覧を表示してユーザーに選択させる
  - 新しいwantreeを作成 → ヒアリング開始
  - 既存のwantreeを編集 → 該当ファイルを開いてヒアリング再開

## ディレクトリ・ファイル操作

MCPを使わず、以下のツールで直接操作する:

### 初期化

```
.wantree/
└── 0/
    └── wantree.yml   ← want:\ntree:\n の初期内容
```

Bashで `.wantree/0/` を作成し、Writeで `wantree.yml` を生成する。

### エントリ追加

既存の連番ディレクトリ（0, 1, 2...）の最大値+1で新しいディレクトリを作成し、
`wantree.yml` を生成する。archiveディレクトリ内の番号も考慮すること。

### 一覧表示

`.wantree/` 内の数字ディレクトリを番号順に読み込み、各 `wantree.yml` の
`want:` フィールドを抽出して表示する。

### アーカイブ

指定エントリのディレクトリを `.wantree/archive/` に移動する（Bashのmvコマンド）。

## ヒアリングフロー

### フェーズ1: 初期化・準備

カレントディレクトリを確認し、必要に応じて `.wantree/` とエントリを作成する。
完了後、ユーザーに伝える:

```
まず、はじめのwantを教えてください。
```

### フェーズ2: root wantの記録

ユーザーが最初のwant（root want）を伝えたら:

1. 音声入力の表記ミスや誤字脱字を自動修正する（意味は変えない）
2. Editツールで `wantree.yml` のwantフィールドを記録する
3. 以下のフォーマットで返答する:

```
（wantree.ymlのコードブロック）

次のwantをどうぞ。
---
終了: x
```

### フェーズ3: ヒアリングの継続

ユーザーがwantを伝えるたびに繰り返す:

1. **入力の判定**:
   - `x` 一文字 → フェーズ4（終了処理）へ
   - 「取り消し」「削除」「消して」などの取り消し指示 → フェーズ3-B（取り消し処理）へ
   - それ以外 → wantとして処理する

2. **wantの追記**:
   - 音声入力の表記ミスや誤字脱字を自動修正する
   - 親子関係が明確な場合は `tree:` を使って階層構造で表現する
   - 複数のwantがカテゴリ別に分別できると判断したら `# コメント` でセクション分けを行う
   - Editツールで `wantree.yml` を更新する

3. **返答**:

```
（更新後のwantree.ymlのコードブロック）

次のwantをどうぞ。
---
終了: x
```

**注意**: ユーザーへの余計な説明や誘導は行わない。返答は上記フォーマットのみ。

### フェーズ3-B: 取り消し処理

ユーザーがwantの取り消しを求めた場合:

1. Readで現在の `wantree.yml` を読み込み、全wantをリストアップする
2. AskUserQuestionで取り消すwantを選択肢として提示する
3. Editでそのwantを削除する
4. 更新後のwantree.ymlをコードブロックで示し、ヒアリングを再開する

### フェーズ4: 終了処理

ユーザーが `x` を送信したら:

「ヒアリングを終了しました。」と伝えて完了する。

## wantree.ymlの記述ルール

- **階層構造**: 親子関係にあるwantは `tree:` を使って表現する
- **セクション分け**: カテゴリ別に分別できる場合は `# コメント` でセクション分けをする
- **誤字修正**: 音声入力の表記ミスや誤字脱字は自動修正するが、意味・内容は変えない
- **入力はそのまま**: 修正が不要な場合はユーザーの入力をそのままwantに使用する

## 注意事項

- **最小限の返答**: ヒアリング中はコードブロック＋「次のwantをどうぞ。」＋終了方法の案内のみ
- **終了方法は毎回表示**: `終了: x` を毎回メッセージに含める
- **MCPは使用しない**: ファイル操作はBash・Write・Editツールで直接行う
